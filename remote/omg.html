Old Mutual Group Corporate Intranet

Background

The purpose of this project was to redevelop an existing SharePoint 2010 publishing site for Old Mutual Group's intranet. I was the lead front-end developer on the project where the primary goal was to speed up client-side performance by:

  • Redesigning the master page and page layout
  • Refactoring main.css and overriding core.css
  • Basic image optimization and css "spriting" techniques

Process

Using Randy Drisgill's minimal master page as a starting point, a lightweight master page containing all global elements (i.e. header, left navigation, content area, and footer) was created. It was a liquid-layout created using divs (elminating out-of-the-box table based layouts) and hid any unused SharePoint controls.

The original build had 9 page layouts. I consolidated these down to 4 - homepage, section overview, content, content aside.

Core.css was also unneccessarily duplicated with custom selectors appended making multiple calls to the server. My approach to branding SharePoint sites is to create a custom stylesheet and incorporate any ms-labelled selectors to override core.css styles affecting my own look and feel.

Basic techniques like optimising PNG images and spriting them ensures that a call to the server only needs to be made once and that file size has been reduced to a minimum. Scaling images down shaved off 150kb per image.
