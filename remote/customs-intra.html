New Zealand Customs Service Intranet Redevelopment

New Zealand Customs Service were using an existing intranet which was essentially an "information dumping ground" - Users were unable to find documents; there was no formal categorisation of information; and a hierarchical site structure extending 9 levels deep.

Approach

I facilitated 2 requirements gathering workshops. Using a cross-technique of affinity diagraming and contextual interviews, we validated each information grouping with the users. I would then asked users to assign a ranking to each group using sticky dots. This helped in identifying user's own priorities against business objectives. The information gained from this session evolved into a requirements document and identified 2 primary personas. Any decisions made needed to align with the values of the personas.

Utilising all the information gained I created a site structure, a taxonomy of metadata, governance model and wireframes with notes. This served as the specification that developers would build towards.

My involvement continued right up to the testing phase providing insights and answering any questions amongst developers, testers and clients.
